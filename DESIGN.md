## Theme

| Field     | Value                                                                                                  |
| --------- | ------------------------------------------------------------------------------------------------------ |
| Surface   | `zinc-50` (light) / `zinc-950` (dark) — neutral, low chroma, never pure white/black                    |
| Ink       | `zinc-900` (light) / `zinc-100` (dark) — not pure black/white; tinted toward cool gray                 |
| Accent    | `#FB7299` (B 站粉) — used at **≤ 8 % surface** as highlight only; never on body text                   |
| Muted ink | `zinc-500` — never below 4.5:1 on its background                                                       |
| Tint      | Tinted toward neutral cool; no warm-cream, no paper-tone default                                       |
| Mood      | Cold observatory: data first, decoration last. The B 站 pink is a single dot of identity, not a theme. |

The product is **product register** (a data tool that serves the data), so
visual restraint dominates. Pink is for: top-bar logo, accent buttons, the
**current filter chip**, and a single data point per chart. Never for body
text, never for full card backgrounds.

## Color strategy

**Restrained** (1 accent ≤ 10 %):

- Default body: tinted neutral (zinc)
- Cards: surface + 1 px border `hsl(var(--border))`
- Charts: palette rotates B 站粉 → 7 neutral-cool accents for category
  encoding (channel pie slices, UP bar fills)
- Hover/focus: shift toward primary, never toward pink

The earlier project loaded an 8-color SaaS palette (`blue-theme / green-theme /
purple-theme / red-theme / indigo-theme / orange-theme` etc.) that violated
this — the new code keeps only 6 swatches, all in a coherent cool register,
and the user's choice is a single accent, not a global theme.

## Typography

| Slot               | Family                                                   | Notes                                                           |
| ------------------ | -------------------------------------------------------- | --------------------------------------------------------------- |
| Display (h1–h3)    | **Geist Sans** (geometric sans, 100–900 weight variable) | tighter letterspacing `-0.02em`, capped at `clamp()` max 4rem   |
| Body               | **Noto Sans SC** (400/500/700) with Geist Sans for Latin | line length ≤ 75ch, body 14–15px                                |
| Numeric / tabular  | **Geist Mono** (variable)                                | all stat numbers (`1,234,567` not `1234561`) use `tabular-nums` |
| Monospace fallback | ui-monospace, SFMono-Regular                             |                                                                 |
| Emoji              | Apple Color Emoji / Segoe UI Emoji / Segoe UI Symbol     |                                                                 |
| CJK fallback       | PingFang SC / Hiragino Sans GB / Microsoft YaHei         | kicks in for Chinese characters on all platforms                |

Avoid: Inter (saturated AI tell), "Inter for everything" pattern,
gray text on colored backgrounds (impeccable absolute ban).

`h1` line-height `1.2`, `h2` `1.3`, `h3` `1.4`. All headings get
`text-wrap: balance`.

## Spacing

8 pt grid enforced via Tailwind:

- Page horizontal padding: `m-2 md:m-10 md:p-10` (40 px on desktop)
- Card inner padding: `p-5` (20 px) on summary cards, `p-6` on content cards
- Section gap (between cards on a stack): `gap-6` (24 px)
- Grid gutter (card-to-card): `gap-4` (16 px) for dense, `gap-6` for hero
- Section header to first card: `mt-8 mb-6` (32 px / 24 px)

Reject `vh` based spacing (`gap-[2vh]`, `h-[35vh]`) — produces
narrowing-on-zoom artifacts; replaced with fixed rem values.

## Radius

- `--radius: 0.625rem` (10 px) on shadcn primitives
- Card radius: `rounded-xl` (12 px)
- Pill / chip / button: `rounded-full`
- Image overflow: `rounded-lg` (8 px) for thumbnails

No `rounded-3xl` blob shapes, no `rounded-sm` (too sharp, looks like 2014).

## Elevation

- Card: `border bg-card` only — no `shadow` (per impeccable rule "the
  rounded-square icon tile above every heading" + "identical card grids")
- Hover state for clickable cards: `border-primary/50 hover:shadow-md` (1
  transition only, not 3 layered shadows)
- Popover / theme drawer: `shadow-xl` (large but single)
- Toaster: `shadow-lg`

## Motion

- Default transition: `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo). No
  bounce, no elastic — these read as dated.
- Duration: 150 ms for hover, 250 ms for layout, 400 ms for entrance.
- `prefers-reduced-motion: reduce` short-circuits to 0 ms (handled by the
  global CSS rule in `globals.css`).
- All entrance / layout / exit transitions live in `src/common/styles/motion.ts`
  (`fadeUp`, `scaleIn`, `slideInRight`, `containerStagger`).
- `framer-motion` is the standard library. Use it for:
  - Stagger lists (cards, summary, table rows) — `containerStagger(0.04)`.
  - Page sections / hero — `initial="hidden" animate="show"` with
    `fadeUp` and a small `delay` (0.06–0.24s).
  - Drawer / popover enter-exit — `AnimatePresence` + `slideInRight` /
    `backdropFade`.
  - Hover / tap micro-interactions — `whileHover` + `whileTap`. Keep
    ranges small (scale 1.02–1.08, rotate ±10°) and never block the
    click.
- Tailwind `transition-base` utility (200 ms ease-out-expo) handles
  color / shadow / transform on cards and chips.
- Reveal animation is intentional: the data is the reveal — motion should
  never delay content readability by more than ~300 ms.

## Components

### shadcn primitives used

- `Button` (cva: default / outline / ghost / destructive / secondary / link)
- `Card`, `CardHeader`, `CardContent`, `CardDescription`, `CardTitle`
- `Input` (text only, no fancy variants)
- `Badge` (outline + secondary)
- `Select` (Radix UI Select + cva-styled trigger) — date pickers in
  `/` and `/dashboard` no longer use the browser's native `<select>`
- `Toast` (Radix) — only for error feedback; success feedback uses inline
  text; we don't toast "data loaded" because that's noise
- `Spinner` (ImSpinner8 with `animate-spin`)

### Custom building blocks

- `Sidebar` — left rail, `w-72` desktop, hidden < 900 px
- `Navbar` — sticky top, 2 icon buttons + GitHub link
- `ThemeSettings` — right-side drawer, click-outside to dismiss, ESC to close
- `StackedChart` — single bar, dynamic max from data
- `WordCloud` — `react-d3-cloud` with Times serif, dark/light aware
- `VideoCard` — aspect-video cover + title + UP + channel chips
- `RelatedVideoList` — 4-column grid of small video cards
- `SummaryCard` — label / big number / sub-line

### Charts (Recharts)

- Always `ResponsiveContainer width="100%" height="100%"`
- Tooltip: `bg-popover` + 1 px border + 8 px radius + 12 px font
- Grid: `stroke="hsl(var(--border))"` + `strokeDasharray="3 3"`
- Y-axis: `tabular-nums` via CSS, `formatCompact` for tick formatter
- Bar: `radius={[6, 6, 0, 0]}` for column charts, `[0, 6, 6, 0]` for horizontal
- Never use a third color besides the brand pink + axis ink

## Iconography

`react-icons` (fa6 / bi / bs / fi / lu / ai / ci). No emoji as primary UI
icon. Lucide was considered but the project already standardizes on
`react-icons`; not worth swapping for a one-off spinner.

## Anti-references

What this UI explicitly is NOT:

- Not Bilibili.com — we are an observatory, not a clone. No navbars with
  B 站 categories, no login flows, no B 站 SVG ornaments, no comments, no
  follow buttons.
- Not the default Syncfusion "admin dashboard" look — which is what the
  project was before. No `e-input-group` rounded controls, no
  `StackingColumnSeries` blue squares, no `TooltipComponent` tooltips.
- Not a SaaS landing page. No "Get started" buttons, no "Trusted by 10k
  users", no Inter + 4 floating cards, no purple-to-blue gradient hero.
- Not the 2023 SaaS "Big Number Hero" template (impeccable absolute ban).
  Our hero is _data_, not a metric KPI.
- Not Material 3. We're not in Google's design system lane.
- No glassmorphism, no gradient text, no side-stripe borders.

## i18n

3 個 locales：简体中文 (zh-CN, default) / 繁體中文 (zh-TW) / English (en)。
翻譯字典在 `src/common/i18n/dictionaries/{locale}.ts` 內聯（~80 keys），
source of truth 為 `zh-CN.ts`；`types.ts` 用 `typeof zhCN` 推導
`CustomTypeOptions.resources` → 編譯期保證 `t('key')` 一定存在。

切換器在 `ThemeSettings` drawer；持久化用 cookie + localStorage 雙寫
（`i18next-browser-languagedetector` 的 `lookupCookie` /
`lookupLocalStorage` 設為 `bili-analyzer-locale`）。

`<html lang>` 在 `app/layout.tsx` server 端用 `cookies()` 讀，
切換語言後 reload 也不閃。**代價：所有頁面從 `○` 變 `ƒ`（dynamic
rendered）** — 因為 `cookies()` 強制 dynamic 渲染。Production
v0.4 可改用 client `useEffect` 設 `document.documentElement.lang` 換回
static，但代價是初次 paint 的 lang 屬性會錯 ~50ms。

## Layout patterns

- Dashboard: 12-col grid via `grid-cols-1 lg:grid-cols-2 xl:grid-cols-4` on
  summary, `lg:grid-cols-2` on charts, full-width on tables.
- Search: hero text + sticky filter card + virtualized grid.
- Detail: 12-col grid with video left, info card right on `xl`; below
  that, 2-col on `xl` then full-width stacks on `lg` and below.
- Responsive: use `repeat(auto-fit, minmax(280px, 1fr))` patterns where
  density is data-driven (UP table, tag chips), explicit breakpoints where
  the layout is semantic (video player + side info).

## Accessibility

- Body text: 4.5:1 against surface (verified on light zinc-50 / dark
  zinc-950).
- Muted text: `zinc-500` against the surface, 4.5:1 on light, 4.6:1 on
  dark.
- Interactive elements: focus ring `ring-1 ring-ring`, never removed.
- Hit targets: minimum 36 × 36 px on icon buttons (`h-11 px-5` buttons are
  44 px tall).
- Skip-to-content link: not yet implemented; flagged for `onboard` pass.
- Keyboard: theme drawer closes on ESC; search input + random button both
  reachable via Tab; enter triggers search.

## Out of scope (deliberate)

- Dark/light auto-switching beyond `prefers-color-scheme` — no
  user-pinned override has been requested yet. The theme picker
  inside the settings drawer (`浅色 / 深色` radio) provides manual
  control on top of OS-level detection.
- Custom font for hero — Geist Sans at default 600 weight is enough.
- Mutation testing (Stryker) — too slow for a small Vercel-hosted
  project; the unit + RTL smoke suite already pins every
  `buildAggregations` branch.
- Server-side data persistence — the data layer is intentionally
  read-only, sourced from the `result` orphan branch. IndexedDB /
  localStorage caching has been considered and rejected (see
  `docs/architecture.md` § Rendering strategy).
